在实验室的服务器上搭建深度学习环境。

1. 安装系统

1. 安装ubuntu

具体安装省略，记录一个小bug，可能在给有独立显卡的台式机安装ubuntu双系统时遇到：在安装时，使用U盘启动这步，直接选择try ubuntu或install ubuntu都会出现黑屏的问题。

解决方法：将光标移动到install ubuntu一项上，按e键，会进入一个可编辑的界面，将quiet splash修改为nouveau.modeset=0 nomodeset，然后按ctrl+x进入安装。之后在ubuntu安装nvidia驱动后，就正常了。如果没有安装驱动，每次进入前，都要用同样的方法将上面的quiet splash修改。

2. 配置nvidia显卡

具体分为两步：

  1. 安装nvidia驱动，如果是图形界面的话，在Software & Updates中的Additional Drivers中选择合适的驱动安装即可。
  2. 在官网下载cudnn并安装。

2. 创建和登录用户

在linux上创建自己的用户，方便管理代码和安装应用。比如我们想要创建一个用户名是haha，密码是123456的用户，命令如下：

  1. 添加用户：useradd -d /home/haha -m haha。
  2. 设置密码(只有设置密码之后，才能登录用户)：passwd haha，然后输入密码。
  3. 然后就可以通过ssh haha@your_ip的方式登录服务器了。登录后也可以设置bash：chsh -s /bin/bash或修改为zsh。
  4. 加入root权限：使用apt下载时，如果出现不在sudoers文件中的报错，则需要将用户加入sudoers，执行sudo vim /etc/sudoers命令，在root ALL=(ALL) ALL的下一行加入haha ALL=(ALL) ALL，然后保存。
  5. 删除用户：userdel -r haha。

3. 安装python

建议直接安装anaconda：

  1. 首先下载安装脚本：wget https://mirrors.tuna.tsinghua.edu.cn/anaconda/archive/Anaconda3-2021.05-Linux-x86_64.sh，
  2. 安装：./Anaconda3-2021.05-Linux-x86_64.sh。
  3. 在~/.bashrc文件的尾部加入alias python=~/anaconda3/bin/python3和alias pip=~/anaconda3/bin/pip，保证我们使用的一定是自己目录下安装的python，而不是用的实验室其他人安装的python。然后执行命令source .bashrc更新配置。
  4. 使用pip或conda安装需要的库。

4. 在服务器上写代码

推荐两种方式：

  1. 使用vscode的插件Remote ssh连接服务器
  2. 使用jupyter写代码。使用jupyter hub。

1. 使用vscode编程

在网上搜vscode的插件Remote ssh。。。

2. 使用jupyter编程

配置服务器上的jupyter，使得在服务器外也能使用jupyter编程。

1. 生成配置文件

jupyter notebook --generate-config

2. 生成密钥

在命令行或python脚本中执行如下代码，然后输入两次密码。

1
2

from notebook.auth import passwd
passwd()

3. 修改配置文件

vim ~/.jupyter/jupyter_notebook_config.py

输入如下内容：

1
2
3
4
5

c.NotebookApp.allow_remote_access = True
c.NotebookApp.ip = '*'
c.NotebookApp.open_browser = False
c.NotebookApp.password = '第二步产生的密钥'
c.NotebookApp.port = 8888

之后就可以在浏览器中输入服务器IP:8888访问服务器上的jupyter，并且使用jupyter直接在服务器上编程了。

如果是使用jupyter lab，则也是类似的操作。

5. 其他

  1. 查看linux版本：cat /proc/version
  2. 查看cpu信息：lscpu
  3. 关闭图形界面：sudo /etc/init.d/gdm3 stop
