Surrogate-Assisted Many-objective Evolutionary Optimization

1. New Challenges in MaOO

Multi-objective时有两到三个目标，many-objective是有四个及以上，many-objective中的挑战是：

  1. 随着目标函数的增多，选择压力减小，被支配的解会变少。
  2. Indicator-based MOEAs中计算复杂度上升。
  3. Decomposition-based搜索效率降低。

解决方法：

  1. 减少目标函数，删除冗余和不重要的目标。
  2. 修改Pareto支配关系，使用$\epsilon$-dominance或$\alpha$-dominance。
  3. 引入额外的criteria。

在multi-objective中的一个基本假设是population-based方法中一个population可以近似一个pareto最优集合，但在many-objective的设定下，由于维度的增大，种群的最大数量限制下，无法准确近似pareto集合。另一个问题是在many-objective问题中，pareto集合的形状可能是不规则的：degenerate(pareto集合的维度小于m-1)、discrete、inverted。所以在many-objective中，使用performance比diversity更加合适。

搜索Knee solutions，有一些指标用于评估搜索knee points和knee region的演化多目标算法：

  1. $KGD = \frac{1}{|S|}\sum_i^{|S|}d(x_i, R)$，评估了收敛性，S为获得的解集合，R为参考点集合，$d(x_i, R)$计算了解$x_i$距离参考解集合R中最近的点的距离。
  2. KIGD则相反，$KIGD = \frac{1}{R}\sum_i^{|R|}d(x_i^r, S)$，更小的KIGD意味着解更加平缓(even)。
  3. $KD = \frac{1}{|K|}\sum_i^{|K|}d(x_i^k, S)$，评估了completeness，$K$为所有knee points的集合。

2. Evolutionary Many-Objective Optimization Algorithms

Reference vector guided many-objective optimization(RVEA)主要解决decomposition-based算法中vector无法在pareto fronts上分布的问题：特点是使用vectors之间的角度而不是欧氏距离测量diversity：

  1. 产生reference vector：首先使用canonical simplex lattice design在unit hyperplane上产生均匀分布的向量，然后normalize。
  2. 然后使用任意的遗传操作生成子代，并且加入种群中，在进行选择之前，需要进行如下操作：
    1. Objective translation：每个个体的每个目标函数值都减掉当前种群中最小的目标函数值。
    2. Population partition：selection操作是在由reference vector划分的子空间中进行，所以要先将所有的个体划分为N(N为reference vector数量)类，划分方法是计算个体到每个reference vector的角度，分入角度最小的类。
    3. Angle penalized distance：对第i个个体，假设属于第j类，则计算惩罚距离：$d_{i, j}(t) = P(\theta_{i, j}(t))||f_i'||, P(\theta_{i, j}(t)) = 1 + m(\frac{t}{t_{max}})^{\alpha}\theta_{i, j}(t)$。t为当前轮数，$t_{max}$为最大轮数，$\alpha$为控制P变化速度的超参数。随着演化的进行，diversity会越来越重要。
  3. Reference vector adaptation：均匀分布的reference vector在很多情况下不会在pareto front上产生均匀分布的解，所以需要加入自适应，$v_i(t+1) = \frac{<v_i(0), f^{max}(t+1) - f^{min}(t+1)>}{v_i(0), f^{max}(t+1) - z^{min}(t+1)>}$，自适应不应该每轮都做，建议每20代做一次。

A Knee-Driven Many-Objective Optimization Algorithm(KnEA)解决dominance-based MOEAs的问题：主要是引入了second criterion。整体框架与NSGA-II类似。

  1. Weighted Distance Based Binary Tournament Mate Selection：选择父代的方法是，首先从父代中随机选择两个个体，如果一个支配另一个，则选择支配的那个；如果无法比较，则判断两个点是否为knee point，如果其中一个是，则选择knee point；如果都不是或都是，则根据weighted distance选择大的有一个；如果weighted distance相等，则随机选一个。 Weighted distance计算方式是利用k 个近邻：$DW = \sum_i^kw_{x_i}dis(x, x_i), w_{x_i} = \frac{r_{x_i}}{\sum_i^kr_{x_i}}, r_{x_i} = \frac{1}{|dis(x, x_i) - \frac{1}{k}\sum_i^k dis(x, x_i)|}$。
  2. Knee Point Based Environment Selection：对于rank相等的，不是通过crowding distance，而是选择knee distance更大的，计算如下：
    1. Constriction of the hyperplane using boundary solutions，使用boundary solution构造超平面。
    2. Knee Point Based Environment Selection

A Two-Archive Algorithm for Many-Objective Optimization：是indicator-based和pareto dominance的方法，将convergence和diversity分别考虑(分为CA和DA两个部分)：

  1. CA阶段使用$I_{\epsilon +}$加速收敛，首先产生子代，然后评估：$I_{\epsilon +} = \min_{\epsilon}(f_i(x_1) - \epsilon \leq f_i(x_2), 1\leq i \leq m), Fitness(x_1) = \sum_{x_2\in P - {x_1}} -e^{-I_{\epsilon +}(x_2, x_1) / 0.05}$，即将fitness向量相距比较近的点赋予比较低的F值。
  2. DA选择diversity最大的解，使用$L_{\frac{1}{m}}$度量，其中m是目标函数的个数。

Corner Sort for Many-Objective Optimization：快速($O(mN^2)$)获得解的非支配关系的一种方法。
